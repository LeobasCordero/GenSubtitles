### Integration Gaps

**Gap #1 — API router bypasses run_pipeline() (Low severity)**  
`api/routers/subtitles.py` does not import or call `run_pipeline`. Instead it manually orchestrates the same steps to reuse the preloaded `WhisperTranscriber` from `app.state`. This is a **structural divergence**: pipeline logic lives in two places. For v1 it's functional, but any future pipeline change must be applied twice.  
Root cause: `run_pipeline()` signature accepts `model_size`/`device` strings and creates `WhisperTranscriber` internally — no parameter for a pre-loaded transcriber.  
Affected: API-01, API-02, API-03, API-04 (all pass, but via different entry point than documented)

**Gap #2 — TRANS-02 same-language guard duplicated inconsistently (Very low severity)**  
`pipeline.py`, `api/routers/subtitles.py`, and `translate_segments()` each implement a same-source-target guard with slight differences. Functionally equivalent. TRANS-02 satisfied in all paths.
