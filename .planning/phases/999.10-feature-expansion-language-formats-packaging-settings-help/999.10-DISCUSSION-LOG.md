## Language Pair Dropdown UX

| Option | Description | Selected |
|--------|-------------|----------|
| Static hardcoded list | Hardcoded display pairs in GUI | |
| Dynamic from API on startup | Populate from GET /languages response | ✓ |
| Two-level: short list + 'More...' | Expandable list | |
| (User freeform) | Source AND target lang dropdowns; all available combinations; error dialog for unsupported combos | — |

**User's choice:** Dynamic from API — both source and target dropdowns populated from `GET /languages`. Source selection filters target dropdown to compatible pairs only. Error dialog on unsupported combination.

**Notes:** User explicitly requested showing source language as a dropdown again (reversed from 999.9 which hid it). Drove two follow-up questions about filtering behavior and dynamic loading.
