# Phase 999.10: Feature Expansion — Language Support, Formats, Settings & Help - Context

**Gathered:** 2026-04-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Seven user-facing capability areas bundled into one expansion:
1. Extended translation language pairs — dynamic source + target language dropdowns from API
2. Additional format support — SSA subtitle output + SSA↔SRT bidirectional conversion (3GP deferred)
3. Standalone portable packaging — **deferred to a future phase**
4. Translation-only mode — GUI tab + CLI subcommand; accepts existing SRT/SSA as input
5. Settings menu — JSON-persisted settings via in-window panel
6. Help menu — Tutorial + language pairs listing + About dialog
7. Menu bar — Native OS menu bar (tkinter.Menu) with Settings and Help menus

Implementation must be done in a **new git branch created from remote** (not from the current local branch head).

</domain>

<decisions>
## Implementation Decisions

### Language Pair Dropdowns
- **D-01:** Re-enable the source language dropdown (hidden in 999.9 — now needed for pair selection). Both source and target language dropdowns are populated **dynamically from `GET /languages` at app startup**. The API response `pairs` list drives both dropdowns — only languages actually available from installed Argos models appear.
- **D-02:** When the user selects a source language, the target language dropdown is **filtered to only show compatible destination languages** for that source pair. This prevents invalid combinations before the user hits Generate.
- **D-03:** If a user somehow reaches Generate with an unsupported pair (e.g., API returns error), **display an error dialog** — not a silent failure.

### Translation-Only Mode
- **D-04:** Translation-only mode is exposed in **both the GUI and CLI**.
- **D-05 (GUI):** A **tab strip or radio at the top of the existing window** switches between "Generate subtitles" mode (current form) and "Translate subtitles" mode (new form). The Translate form accepts a SRT or SSA file as input (instead of video), with source + target language dropdowns and output path.
- **D-06 (CLI):** A new CLI subcommand: `gensubtitles translate --input file.srt --target-lang es` (or similar).
- **D-07:** Output format **preserves input format** — translating a .srt produces a .srt; translating a .ssa produces a .ssa.

### Format Support (SSA Output)
- **D-08:** SSA output format is selected via a **GUI dropdown** in the main form (and translate form) — options: SRT / SSA. The output format dropdown replaces the hardcoded assumption that output is always .srt.
- **D-09:** SSA↔SRT bidirectional conversion is available in **both CLI and GUI**: CLI subcommand `gensubtitles convert input.ssa output.srt` (and reverse direction); GUI exposes conversion via the "Translate subtitles" tab (a "Convert only" option with no language change).

### Format Support (3GP Input)
- **D-10 (deferred):** 3GP video input support is **skipped for this phase** — noting for future backlog.

### Portable Packaging
- **D-11 (deferred):** Standalone packaging is **deferred to a future phase** — the feature set in this phase is already substantial.

### Menu Bar
- **D-12:** Use **native OS menu bar via `tkinter.Menu`** attached to the CTk window root. This provides the standard OS menu bar look on all platforms.
- **D-13:** Menu bar has **two top-level menus: Settings and Help**. No File menu.

### Settings Persistence
- **D-14:** Settings are persisted to a **JSON file in the user config directory** (e.g., using `platformdirs` → `user_config_dir("GenSubtitles")/settings.json`). Settings survive app restarts.
- **D-15:** Settings UI is presented as an **in-window panel that replaces main content** (not a modal dialog). User navigates to Settings, changes values, and returns to the main form.
- **D-16:** Appearance mode setting exposes **three options: Light / Dark / System** — mapped to `ctk.set_appearance_mode()`.
- **D-17:** UI language setting supports **Spanish and English only** in this phase.

### Help Menu
- **D-18:** Help menu includes all three items: **Tutorial walkthrough + language pairs listing + About dialog**. The language pairs listing is dynamic (from `GET /languages`). Tutorial is static text in a CTkToplevel window. About shows app version and license.

### Agent's Discretion
- Exact tab widget implementation for the mode toggle (CTkTabview vs CTkSegmentedButton vs radio buttons).
- Whether "Translate subtitles" tab and "Generate subtitles" tab share language dropdowns or each have their own instances.
- Whether `gensubtitles convert` is a separate subcommand or a flag on an existing command.
- SSA parsing library choice (the `ass` library, or manual parsing — agent to evaluate).
- The exact `platformdirs` call pattern for the settings file path.
- Whether Settings in-window panel uses `.pack_forget()`/`.pack()` or grid switching to replace content.

</decisions>

<specifics>
## Specific Ideas

- **Language pairs:** User wants the combination selector UX — source picks constrain target options dynamically, not just a flat list. API drives it.
- **Settings panel:** In-window replacement (not modal) was explicitly chosen — creates a more app-like feel with full-window settings.
- **SSA conversion in GUI:** The "Translate subtitles" tab doubles as a conversion tool — "Convert only" option (no language change) surfaces the SSA↔SRT conversion in the GUI.

</specifics>

<deferred>
## Deferred Ideas

- **Standalone portable packaging** — Single executable for Windows/macOS/Linux. Deferred by user — phase is already large without it. Note for future 999.x backlog phase.
- **3GP video input** — Trivial FFmpeg extension but skipped in favor of other format work. Future backlog item.
- **Additional color themes** — Color theme selector (beyond Light/Dark/System) mentioned but not chosen. Future backlog if needed.

</deferred>
