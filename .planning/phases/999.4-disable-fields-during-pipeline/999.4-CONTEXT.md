# Phase 999.4: GUI — Disable Fields During Pipeline - Context

**Gathered:** 2026-04-10
**Status:** Ready for planning

<domain>
## Phase Boundary

While the subtitle generation pipeline is running, disable all interactive input controls to prevent mid-run modifications. Re-enable them when generation completes (success or error). The Generate and Clear buttons are already handled — this phase adds the 4 text entry fields and 2 browse/save buttons to the disable/re-enable cycle.

</domain>

<decisions>
## Implementation Decisions

### Entry Field Storage
- **D-01:** Store the 4 `CTkEntry` widgets as named instance attributes during `_build_ui`:
  - `self._entry_input` (Row 0 input video entry)
  - `self._entry_output` (Row 1 output SRT entry)
  - `self._entry_source_lang` (Row 2 source language entry)
  - `self._entry_target_lang` (Row 3 target language entry)
  
  Each entry is assigned immediately after creation, before `.grid()` is called. Consistent with `self._btn_generate` and `self._btn_clear` pattern.

### Browse/Save Button Storage
- **D-02:** Store the 2 browse/save buttons as named instance attributes during `_build_ui`:
  - `self._btn_browse_input` (Row 0 "Browse…" button)
  - `self._btn_browse_output` (Row 1 "Save as…" button)
  
  Consistent with D-01 and the existing named button pattern.

### Disable/Re-enable Scope
- **D-03:** All 6 new widgets are disabled in `_on_generate` and re-enabled in `_finish_generate`:
  - `self._entry_input.configure(state="disabled")`
  - `self._entry_output.configure(state="disabled")`
  - `self._entry_source_lang.configure(state="disabled")`
  - `self._entry_target_lang.configure(state="disabled")`
  - `self._btn_browse_input.configure(state="disabled")`
  - `self._btn_browse_output.configure(state="disabled")`
  
  Re-enable in `_finish_generate` uses `state="normal"` for all 6. The Clear button re-enable remains via `self._update_clear_state()` (existing, unchanged).

### Agent's Discretion
- Whether to extract a helper method (e.g., `_set_inputs_enabled(True/False)`) or inline the 6 `.configure()` calls directly in `_on_generate` and `_finish_generate`.

</decisions>
