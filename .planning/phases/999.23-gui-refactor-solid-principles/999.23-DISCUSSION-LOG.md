## Code Scout Findings

Agent read `main.py` lines 1888–2031 to map the extraction boundary precisely.

**Extracts cleanly (widget-free):**
- `_start_server()` — uvicorn config + thread creation + `_wait_for_server` polling loop
- `_stop_server()` — single-line `server.should_exit = True`
- Module-level constants: `_SERVER_PORT`, `_BASE_URL`, `_SERVER_BIND_TIMEOUT`
- `self._server` instance variable

**Stays in `GenSubtitlesApp` (widget-touching):**
- `_apply_startup_progress(message, progress)` — writes to `self._progress_bar` and `self._stage_label`
- `_on_server_ready()` — hides progress, enables `self._btn_generate` / `self._tl_btn_translate`, calls `_populate_language_dropdowns`
- `_on_server_failed(detail)` — writes red error to progress/label widgets
- `on_closing()` — will call `server.stop()` instead of `self._stop_server()` after refactor

**Key design insight (from scout):**
The polling loop inside `_wait_for_server` currently reads `self._closing` directly to know
when to abort. This is the one hidden coupling that needs to be replaced. Solution: `server.start()`
accepts an `is_closing: Callable[[], bool]` argument — `GenSubtitlesApp` passes
`lambda: self._closing`. This keeps server.py fully widget-free.
