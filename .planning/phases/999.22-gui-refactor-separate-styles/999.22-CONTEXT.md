# Phase 999.22: REFACTOR — Separate GUI Styles from Components - Context

**Gathered:** 2026-04-14
**Status:** Ready for planning

<domain>
## Phase Boundary

Extract all styling definitions — paddings, widget dimensions, and widget color/font
configurations — out of `GenSubtitlesApp` in `gensubtitles/gui/main.py` into a new
dedicated module `gensubtitles/gui/styles.py`. Update all widget constructors and
`_apply_theme()` to use helper functions from `styles.py`.

Scoped to `gensubtitles/gui/` only. No API, pipeline, CLI, or settings changes.

**Important narrowing from Phase 999.21 (D-08):** `theme.py` already owns colors (`p()`)
and typography (`font()`). Phase 999.22 adds **spacing constants**, **dimension constants**,
and **widget-style helper functions** — not a second color system.

</domain>

<decisions>
## Implementation Decisions

### New Module
- **D-01:** Create `gensubtitles/gui/styles.py` as a new file, separate from `theme.py`.
  `theme.py` remains color+typography only. `styles.py` owns layout values and
  widget-style application helpers.

### Module Shape
- **D-02:** `styles.py` exposes **helper functions** that call `widget.configure(...)` directly.
  Functions import `p` and `font` from `.theme` at module level and use them inside
  each helper so colors are resolved at call time (not at import time — avoids capturing
  the wrong appearance mode).

  Expected helpers (names are illustrative — planner finalises exact names):
  - `apply_entry_style(widget)` — sets `fg_color=p("input_bg"), text_color=p("text_primary")`
  - `apply_accent_btn_style(widget)` — sets `fg_color=p("accent"), hover_color=p("accent_hov")`
  - `apply_secondary_btn_style(widget)` — sets `fg_color=p("secondary"), hover_color=p("secondary_hov")`
  - `apply_cancel_btn_style(widget)` — sets `fg_color=p("progress_err"), ...`
  - `apply_progress_bar_style(widget)` — sets `progress_color=p("progress_idle")`
  - `apply_stage_label_style(widget)` — sets `text_color=p("text_secondary")`

  `styles.py` also exports **spacing and dimension constants** for values that appear
  in `.grid()` / `.pack()` calls and widget constructors:
  - `SPACING_XS`, `SPACING_SM`, `SPACING_MD`, `SPACING_LG`, `SPACING_XL` covering the
    recurring values (4, 8, 12, 16, 24, 32) observed in `main.py`
  - `BTN_HEIGHT_PRIMARY = 44` (Generate, Translate, Save, Back, Clear buttons)
  - `BTN_HEIGHT_CANCEL = 36` (Cancel button)
  - `BTN_HEIGHT_MINI = 28` (color swatch buttons)
  - `BTN_WIDTH_BROWSE = 80` (Browse…/Save as… buttons)
  - `BTN_WIDTH_NARROW = 100` (other small buttons)
  - `PROGRESS_BAR_HEIGHT = 16`

### Call Site Pattern
- **D-04:** **Apply-after-construction** — create the widget with its functional/behavioral
  args, then call the style helper on the next line:
  ```python
  self._entry_input = ctk.CTkEntry(self._frame, textvariable=self._input_var)
  apply_entry_style(self._entry_input)
  ```
  Constructors keep: parent, textvariable/command/variable/values, state, text_color_disabled,
  grid/pack geometry args. Constructors lose: fg_color, hover_color, text_color, font,
  height (when covered by a constant), width (when covered by a constant).

### `_apply_theme()` Simplification
- **D-03:** `_apply_theme()` in `main.py` currently duplicates every color assignment from
  the constructors. After the refactor it must call the **same helper functions** from
  `styles.py` instead of inline `configure()` calls. The method body should shrink
  significantly — one helper call per widget/group replaces the repeated
  `entry.configure(fg_color=..., text_color=...)` blocks.

### Agent's Discretion
- Exact constant names (`SPACING_MD` vs `PAD_MD` vs `PAD_16`) — follow the most
  readable naming given the codebase style.
- Whether to group constants into a `Spacing` / `Sizes` namespace or keep flat module-level names.
- Whether helper functions accept an optional extra-kwargs `**kw` parameter for
  one-off overrides without duplicating a whole helper.
- Exact set of helper functions — add more or merge if the widget survey reveals
  additional roles not listed above.

</decisions>

## Existing Code Insights

### Reusable Assets
- `gensubtitles/gui/theme.py` — `p(key)` and `font(role)` already exported; helpers in
  `styles.py` will import and call these.

### Established Patterns
- `theme.py` module pattern: module-level import of `customtkinter as ctk`, private dicts,
  public functions. `styles.py` should follow the same pattern.
- Apply-after-construction is already used for grid/pack geometry — extending it to theme
  helpers is consistent with existing style.

### Integration Points
- `main.py` imports: add `from .styles import apply_entry_style, apply_accent_btn_style, ...`
  alongside existing `from .theme import font, p`.
- `_apply_theme()` calls in `sync_with_os()` / `_apply_startup_theme()` — these call
  `_apply_theme()`; the method's signature doesn't change, only its body.
