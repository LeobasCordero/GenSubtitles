## A — What lives in `styles.py`

| Option | Description | Selected |
|--------|-------------|----------|
| Extend `theme.py` | Add spacing/sizing constants and widget-style dicts to existing theme module | |
| New `styles.py` | Separate file for layout concerns; `theme.py` stays color+typography only | ✓ |
| New `styles.py` + absorb color combos | Widget-config dicts bundling fg_color+hover_color | |

**User's choice:** New `styles.py` — separated from `theme.py`
**Notes:** `theme.py` owns colors and typography; `styles.py` owns spacing, dimensions, and widget-style helpers.

---

## B — Module shape

| Option | Description | Selected |
|--------|-------------|----------|
| Named constants only | Flat module-level names like `SPACING_MD = 16`, `BTN_HEIGHT = 44` | |
| Config dicts per widget role | `ENTRY_STYLE = {"padx": 8}` unpacked with `**` | |
| Helper functions | `apply_entry_style(widget)` calls `widget.configure(...)` directly | ✓ |

**User's choice:** Helper functions
**Notes:** Since colors from `p()` must be resolved at call time (not import time), helper functions that call `widget.configure()` directly are the correct pattern. Constants for spacing/sizes are also included.

---

## C — `_apply_theme()` fate

| Option | Description | Selected |
|--------|-------------|----------|
| Simplify `_apply_theme()` | Replace duplicated inline `configure()` calls with helper function calls | ✓ |
| Remove `_apply_theme()` entirely | Call helpers from `sync_with_os()` directly | |
| Leave alone | Only touch constructors this phase | |

**User's choice:** Simplify
**Notes:** `_apply_theme()` signature stays; body is replaced with helper calls from `styles.py`.

---

## D — Call site strategy

| Option | Description | Selected |
|--------|-------------|----------|
| Apply after construction | `self._entry = ctk.CTkEntry(...); apply_entry_style(self._entry)` | ✓ |
| Pass style kwargs inline | `self._entry = ctk.CTkEntry(..., **entry_style())` | |
| Agent's discretion | Leave pattern to planner | |

**User's choice:** Apply after construction
**Notes:** Constructor keeps functional args (textvariable, command, state, etc.). Style helper called on the next line.
