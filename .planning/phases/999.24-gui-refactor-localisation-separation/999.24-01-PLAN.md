<objective>
Extract the `_STRINGS` dict and lookup logic from `GenSubtitlesApp` into a new `gensubtitles/gui/locale.py` module. Update all 66 call sites from `self._s(key)` to the module-level `s(key)`. This completes the GUI refactor series: `theme.py` (colors/typography) → `styles.py` (spacing/dimensions) → `locale.py` (localisation strings).

Purpose: Decouple localisation data from widget logic, making it trivial to add new language pairs without touching GUI component code.

Output: New `locale.py` module + slimmed `main.py` with no inline string registry.
</objective>

Create `gensubtitles/gui/locale.py` following the `theme.py` / `styles.py` pattern. The module:

1. **Docstring** — match style of theme.py/styles.py: module name, description, Public API section listing `s(key)`, `set_language(lang)`, `s_lang(key, lang)`.

2. **Move `_STRINGS` unmodified** from `main.py` — copy the entire dict as-is (both "en" and "es" sub-dicts, ~170 lines). Keep it private (underscore-prefixed).

3. **Add `LANGUAGES` tuple** — `LANGUAGES: tuple[str, ...] = tuple(_STRINGS.keys())`. This is the public iterable of supported language codes; `main.py` uses it to avoid exposing `_STRINGS` directly.

4. **Add `_active_lang` module state** — `_active_lang: str = "en"` — initialized to English; no settings import.

5. **Implement `set_language(lang: str) -> None`** — updates `_active_lang`. Accepts any string (falls back gracefully in `s()` if unknown). No validation needed — the caller is responsible for passing a valid locale code.

6. **Implement `s(key: str) -> str`** — the primary public lookup. Replicates the existing `_s()` fallback logic but reads `_active_lang` instead of `self._current_settings`:
   ```python
   def s(key: str) -> str:
       locale_dict = _STRINGS.get(_active_lang, _STRINGS["en"])
       return locale_dict.get(key, _STRINGS["en"].get(key, key))
   ```

7. **Implement `s_lang(key: str, lang: str) -> str`** — lang-specific lookup without changing `_active_lang`. Used by `_relabel_tabview_headers()` in main.py to look up Spanish tab names as map keys:
   ```python
   def s_lang(key: str, lang: str) -> str:
       locale_dict = _STRINGS.get(lang, _STRINGS["en"])
       return locale_dict.get(key, _STRINGS["en"].get(key, key))
   ```

File layout order: docstring → `from __future__ import annotations` → `_STRINGS` → `LANGUAGES` → `_active_lang` → `set_language` → `s` → `s_lang`.
