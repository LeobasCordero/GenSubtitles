## What Was Built

Extracted the `_STRINGS` dict and lookup logic from `GenSubtitlesApp` in `main.py` into a new `gensubtitles/gui/locale.py` module, following the same pattern as `theme.py` (exports `p`, `font`) and `styles.py`.

### New File: `gensubtitles/gui/locale.py`

- `_STRINGS` dict (private) — complete English + Spanish string registry (~170 lines)
- `LANGUAGES: tuple[str, ...]` — public iterable of supported language codes
- `_active_lang: str` — module-level state (default `"en"`)
- `set_language(lang)` — update active language
- `s(key)` — primary lookup using active language with English fallback
- `s_lang(key, lang)` — language-specific lookup without changing state

### Updated: `gensubtitles/gui/main.py`

- Added `from .locale import s, set_language, s_lang, LANGUAGES`
- Removed `_STRINGS` dict (~170 lines deleted)
- Removed `_s()` method
- Replaced all 66 `self._s(key)` calls with `s(key)`
- `_build_ui`: replaced `_STRINGS.get(...)` startup stage text with `set_language()` + `s()`
- `_relabel_tabview_headers()`: replaced `_STRINGS["es"][...]` with `s_lang(..., "es")`
- `_apply_ui_language()`: added `set_language(lang)` at top; replaced `_STRINGS` iterations with `s_lang(key, lang_code) for lang_code in LANGUAGES`
- `_save_settings()`: added `set_language(new_settings.ui_language)` before `_apply_ui_language()`
