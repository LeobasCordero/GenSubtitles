# Phase 999.13: Subtitle Style Settings - Context

**Gathered:** 2026-04-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Allow users to configure the visual appearance of generated subtitles — font family, font size, text color, and outline color — stored in AppSettings and applied to SSA output via pysubs2. Style settings have no effect on SRT output.

</domain>

<decisions>
## Implementation Decisions

### Color Input (D-01)
- **D-01:** Use native OS color picker — `tkinter.colorchooser.askcolor()`. Each color field (text color, outline color) gets a small colored swatch button that opens the OS dialog. Result stored as hex string (e.g. `#FFFFFF`).

### Font Family (D-02)
- **D-02:** `CTkOptionMenu` dropdown of smart TV-compatible fonts only: Arial, Helvetica, Verdana, Trebuchet MS, Tahoma. No free-text entry — avoids invalid font names that silently fall back in pysubs2.

### Settings Panel Layout (D-03)
- **D-03:** Add a "Subtitle Style" subsection separator label inside `_build_settings_panel()` between the existing general settings (rows 1–3) and the new style fields (rows 5–9). Use a `CTkLabel` with a lighter text color as a visual divider at row 4.

### SRT Scope (D-04)
- **D-04:** Style settings apply to SSA output only. SRT output is unaffected — no inline HTML tags. The Settings panel shows style fields regardless of selected output format (they simply take effect only when SSA is produced).

### Agent's Discretion
- Default values for the 4 style fields: match pysubs2 SSA defaults (Arial, size 20, white `#FFFFFF` text, black `#000000` outline, outline width 2). These are the values `write_ssa()` currently produces implicitly.
- Font size input: integer `CTkEntry` or small spinner — agent chooses whichever fits the settings panel layout cleanly.
- Outline width: agent decides whether to expose as a setting or hardcode at 2 (the SSA default). Keep scope lean — expose only if trivial to add.
- Localization: add all new label keys to both `_STRINGS["en"]` and `_STRINGS["es"]` dicts in `gui/main.py`; don't add a Settings tutorial note about SSA-only scope unless there's a natural place for it.

</decisions>

<canonical_refs>
## Canonical References

**Downstream agents MUST read these before planning or implementing.**

### Settings persistence
- `gensubtitles/core/settings.py` — `AppSettings` dataclass, `load_settings()`, `save_settings()`. New style fields go here as flat fields (e.g. `subtitle_font_family`, `subtitle_font_size`, `subtitle_text_color`, `subtitle_outline_color`).

### SSA output
- `gensubtitles/core/srt_writer.py` — `write_ssa()` uses `pysubs2.SSAFile()` + `pysubs2.SSAEvent`. Style is applied via `subs.styles["Default"]` (a `pysubs2.SSAStyle` object). Fields: `fontname`, `fontsize`, `primarycolor` (pysubs2 Color), `outlinecolor` (pysubs2 Color), `outline` (float, width).

### GUI settings panel
- `gensubtitles/gui/main.py` — `_build_settings_panel()` (row grid, existing fields), `_save_settings()` (constructs new AppSettings), `_show_settings()` (populates fields from current_settings), `_apply_ui_language()` (re-labels settings widgets). All three methods must be updated.

### Localization
- `gensubtitles/gui/main.py` — `_STRINGS` dict at the top of the file. New label keys for the subtitle style section must be added to both `"en"` and `"es"` entries.

</canonical_refs>

<code_context>
## Existing Code Insights

### Reusable Assets
- `tkinter.colorchooser.askcolor()` — built-in, no new dependency. Returns `((r,g,b), "#rrggbb")` or `(None, None)` on cancel.
- `pysubs2.SSAStyle` — accessible via `subs.styles["Default"]`; fields `fontname` (str), `fontsize` (int), `primarycolor` / `outlinecolor` (pysubs2.Color), `outline` (float).
- `pysubs2.Color.from_rgb(r, g, b)` — converts RGB to pysubs2 Color (needed since we store hex in AppSettings).
- `ctk.CTkOptionMenu` — already used for font family in existing dropdown controls.

### Established Patterns
- Settings panel uses 2-column grid (label col=0, control col=1) with `sf.columnconfigure(1, weight=1)`. New rows append sequentially.
- All new `CTkLabel` and button references must be stored as `self._lbl_*` / `self._btn_*` on the instance so `_apply_ui_language()` can relabel them.
- `_save_settings()` constructs a fresh `AppSettings(...)` explicitly naming all fields — must include new style fields and carry over existing ones.
- `load_settings()` uses key filtering: new fields with defaults are backward-compatible (old JSON without the keys will use the dataclass defaults).

### Integration Points
- `write_ssa()` in `srt_writer.py` must accept an optional `style` parameter (or a dedicated style dataclass/dict) so the pipeline can pass user settings through.
- `run_pipeline()` in `pipeline.py` calls `write_ssa()` indirectly — may need a `subtitle_style` kwarg forwarded from CLI/API/GUI.

</code_context>

<specifics>
## Specific Ideas

- Font dropdown values (smart TV-compatible): `["Arial", "Helvetica", "Verdana", "Trebuchet MS", "Tahoma"]`
- Color swatch button: a small `ctk.CTkButton` (width ~36, height ~28) whose `fg_color` shows the current stored color. Clicking it opens `askcolor(color=current_hex)` so the dialog opens pre-filled.
- On cancel from color picker: keep the previous value (don't reset to default).

</specifics>
