# Phase 999.11: Subtitle Silence — VAD & Timestamp Quality - Discussion Log

> **Audit trail only.** Do not use as input to planning, research, or execution agents.
> Decisions are captured in CONTEXT.md — this log preserves the alternatives considered.

**Date:** 2026-04-11

---

## Area 1: VAD Parameter Exposure

**Question:** How should the tuned VAD parameters be implemented?  
**Options presented:**
- Hardcode in transcriber.py ← **selected**
- Expose as optional kwargs (API-ready but not wired up)
- Fully expose in GUI/API/CLI surface

**Decision:** Hardcode the recommended values (`min_silence_duration_ms=400`, `speech_pad_ms=200`, `min_speech_duration_ms=250`) directly in `transcriber.py`. No new API/CLI/GUI surface added.

---

## Area 2: Word Timestamps Usage

**Question:** What do we do with `word_timestamps=True`?  
**Options presented:**
- Tighten segment end-times only ← **selected**
- Tighten + auto-split long segments
- Collect only, don't use for timing

**Decision:** Enable `word_timestamps=True`. Use `seg.words[-1].end` to replace each segment's end-time. Per-word data is not surfaced in output or SRT.

---

## Area 3: Model Size Default

**Question:** Change the default Whisper model size?  
**Options presented:**
- Keep small
- Bump to medium ← **selected**
- Keep small + add hint in UI

**Decision:** Change default to `"medium"` in both `WhisperTranscriber.__init__()` and `transcribe_audio()`.

**Follow-up:** First-run download impact (~1.5GB).  
**Options presented:**
- Default only, no other surface changes
- Default + update README/help text ← **selected**

**Decision:** Update `README.md`, `README.es.md`, and GUI Help dialog with a note about the larger first-run download and how to use `--model small`.

---

## Area 4: SRT Output Impact (End-Time Adjustment Location)

**Question:** Where should the word-timestamp end-time adjustment happen?  
**Options presented:**
- Patch end-time in transcriber.py ← **selected**
- Handle in srt_writer.py
- Post-process in pipeline.py

**Decision:** Patch in `transcriber.py` after `list(segments_gen)`, before returning `TranscriptionResult`. Keeps all consumers unchanged.

**Follow-up:** What if a segment has no words?  
**Options presented:**
- Fall back to seg.end when no words
- Drop wordless segments ← **selected**

**Decision:** Drop (filter out) wordless segments entirely. They represent silence-only artifacts with no speech content.
