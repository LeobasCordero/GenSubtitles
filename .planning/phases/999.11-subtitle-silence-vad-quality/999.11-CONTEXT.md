# Phase 999.11: Subtitle Silence — VAD & Timestamp Quality - Context

**Gathered:** 2026-04-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Tighten subtitle timing by tuning faster-whisper's VAD silence detection parameters and enabling
word-level timestamps so each segment ends exactly where speech ends rather than at a padded chunk
boundary. Default model size bumped from `small` to `medium` for improved timestamp precision.

**Scope:** Changes to `gensubtitles/core/transcriber.py`, default model size, and README/help text.
No new CLI flags, API params, or GUI controls for VAD values.

</domain>

<decisions>
## Implementation Decisions

### VAD Parameter Tuning
- **D-01:** Hardcode the tuned VAD values directly in `transcriber.py` — no API/CLI/GUI knobs.
  Values to use (per ROADMAP context):
  - `min_silence_duration_ms = 400` (down from the 2000ms default)
  - `speech_pad_ms = 200`
  - `min_speech_duration_ms = 250`
  Pass as `vad_parameters={"min_silence_duration_ms": 400, "speech_pad_ms": 200, "min_speech_duration_ms": 250}` in the `transcribe()` kwargs dict alongside `vad_filter=True`.

### Word Timestamps
- **D-02:** Enable `word_timestamps=True` in the `transcribe()` kwargs.
- **D-03:** Use the word timestamps **only** to tighten segment end-times. After materialising
  segments, replace each segment's `.end` with `seg.words[-1].end` when `.words` is non-empty.
  This keeps the SRT writer/pipeline untouched — they see normal-looking segments.
- **D-04:** Segments with an empty or missing `.words` list **must be dropped** from the output
  (not passed to the SRT writer). These are silence-only segments that slipped through VAD and
  have no real speech content.
- **D-05:** Patching happens in `WhisperTranscriber.transcribe()` after `list(segments_gen)` is
  called — before `TranscriptionResult` is returned. No changes to `srt_writer.py` or `pipeline.py`.

### Model Size Default
- **D-06:** Change default `model_size` in both `WhisperTranscriber.__init__()` and
  `transcribe_audio()` from `"small"` to `"medium"`. The `--model` CLI flag and API parameter
  remain unchanged so users can still downgrade.
- **D-07:** Update README (both `README.md` and `README.es.md`) and the GUI Help dialog
  (if applicable) to mention that the medium model (~1.5 GB first-run download) is the new
  default and that users can pass `--model small` for a lighter download.

### the agent's Discretion
- Exact wording of the README warning sentence
- Whether to add a `# TRN-xx` comment code for the VAD parameters block (follow existing `TRN-0x` pattern)
- Whether the namedtuple `TranscriptionResult.segments` field documentation needs updating given
  words are now consumed internally (they are NOT surfaced in the result)

</decisions>

- Segment `.words` is `None` (not `[]`) when `word_timestamps=False`; when `True` it is always
  a list (may be empty for zero-duration segments)

<code_context>
## Existing Code Insights

### Reusable Assets
- `WhisperTranscriber.transcribe()` — the single place to add `vad_parameters` and `word_timestamps`; patch end-times here after `list(segments_gen)`.
- `TranscriptionResult` namedtuple — unchanged; words are consumed internally, not exposed.
- `VALID_MODEL_SIZES` frozenset — already contains `"medium"`; no change needed there.

### Established Patterns
- `kwargs` dict is built in `transcribe()` before calling `self.model.transcribe()` — extend it the same way (`kwargs["vad_parameters"] = {...}` and `kwargs["word_timestamps"] = True`).
- `list(segments_gen)` materialises the generator (TRN-05 pattern) — iterate the resulting list to patch end-times before building `TranscriptionResult`.
- Comment codes follow `TRN-NN` convention — new lines should be annotated consistently.

### Integration Points
- `pipeline.py` calls `WhisperTranscriber.transcribe()` — receives `TranscriptionResult`; no changes needed there since the patching is transparent.
- `srt_writer.py` reads `seg.start` / `seg.end` — untouched.
- `tests/test_transcriber.py` TRN-04 test mocks `model.transcribe()` — will need a companion TRN-06/TRN-07 test for the new kwargs.

</code_context>
