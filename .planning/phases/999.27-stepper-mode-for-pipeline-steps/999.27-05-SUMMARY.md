## What Was Built

Added the GUI stepper widget to `gensubtitles/gui/main.py`.

### New Instance Variables (in `__init__`)
- `self._work_dir_var: ctk.StringVar` — bound to work directory entry
- `self._step_states: dict[str, str]` — per-step state ("pending"/"running"/"done"/"error")
- `self._stepper_refresh_id: str | None` — `after()` cancel token for polling loop

### New UI: rows 12-15 in Generate tab
- **Row 12**: "Step-by-step mode" section label (secondary style)
- **Row 13**: Work directory entry + Browse button
- **Row 14**: Stepper frame — 4 stage columns (Extract Audio / Transcribe / Translate / Write SRT), each showing a status label and an action button  
- **Row 15**: "Clear Work Files" button (disabled until artifacts exist)

### New Methods
| Method | Purpose |
|--------|---------|
| `_build_stepper_section()` | Constructs rows 12-15 |
| `_browse_work_dir()` | Opens directory picker |
| `_schedule_stepper_refresh()` | Schedules 2-second polling loop |
| `_do_stepper_refresh()` | Called each tick; re-schedules |
| `_refresh_stepper_state()` | Checks file existence, enables/disables buttons |
| `_run_step_in_bg(step_key, endpoint, payload)` | Background thread runner for step API calls |
| `_on_step_success(step_key)` | Updates state to "done" on main thread |
| `_on_step_error(step_key, detail)` | Updates state to "error", shows detail in stage label |
| `_on_step_extract()` | Calls POST /steps/extract |
| `_on_step_transcribe()` | Calls POST /steps/transcribe |
| `_on_step_translate()` | Calls POST /steps/translate |
| `_on_step_write()` | Calls POST /steps/write |
| `_on_clear_work()` | Deletes 3 artifacts, resets step states |

### Modified: `on_closing()`
Added `after_cancel(self._stepper_refresh_id)` before server stop to prevent timer callbacks after window destruction.

## Key Decisions
- Button enable logic: extract requires input video + server ready; transcribe requires audio.wav; translate requires transcription.json + target lang selected; write requires transcription.json or translation.json
- Translate shows "⊘ Skipped" when no target language configured (per D-08)
- `_run_step_in_bg` uses `requests.post(..., timeout=600)` — consistent with other API calls
- Polling interval: 2 seconds (matches plan spec)
