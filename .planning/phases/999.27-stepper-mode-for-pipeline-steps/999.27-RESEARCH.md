## Key Architecture Decisions

### 1. New module: `gensubtitles/core/steps.py`

`run_pipeline()` in `pipeline.py` uses `audio_temp_context()` which **auto-deletes** the WAV
after the context exits. Step mode must persist the WAV in `work_dir` ‚Äî cannot use
`audio_temp_context()`. A new module is cleaner than modifying `pipeline.py`.

### 2. `transcribe_step()` accepts optional pre-loaded `WhisperTranscriber`

```python
def transcribe_step(
    work_dir: Path,
    transcriber: Optional[WhisperTranscriber] = None,
    model_size: str = "medium",
    source_lang: Optional[str] = None,
    device: str = "auto",
) -> TranscriptionResult: ...
```

- **CLI path**: `transcriber=None` ‚Üí step lazily creates its own (matching `pipeline.py` pattern)
- **API path-based endpoint**: `transcriber=request.app.state.transcriber` (reuse pre-loaded model)

### 3. JSON segment schema

`faster_whisper.transcribe.Segment` has many fields. Serialize only `start`, `end`, `text` (D-06).
Deserialize via `collections.namedtuple("_Segment", ["start", "end", "text"])`.
Both `write_srt()` and `translate_segments()` are duck-typed ‚Äî only need `.start`, `.end`, `.text`.

### 4. Two API endpoint families

**Upload/download (D-17/D-18)** ‚Äî for external API consumers:
- `POST /subtitles/extract` ‚Äî video UploadFile ‚Üí WAV file response
- `POST /subtitles/transcribe` ‚Äî WAV UploadFile ‚Üí `transcription.json` response
- `POST /subtitles/translate` ‚Äî segments JSON UploadFile + query params ‚Üí `translation.json` response
- `POST /subtitles/write` ‚Äî segments JSON UploadFile ‚Üí SRT response

Added to existing `api/routers/subtitles.py`. Not suitable for GUI: video files can be multiple GB
(in-memory upload is too expensive).

**Path-based (for GUI)** ‚Äî local-machine use only, new `api/routers/steps.py`:
- `POST /steps/extract` ‚Äî JSON `{video_path, work_dir}` ‚Üí `{status, output_path}`
- `POST /steps/transcribe` ‚Äî JSON `{work_dir, model_size?, source_lang?, device?}` ‚Üí `{status, output_path}`
- `POST /steps/translate` ‚Äî JSON `{work_dir, target_lang, engine?}` ‚Üí `{status, output_path}`
- `POST /steps/write` ‚Äî JSON `{work_dir, output_path}` ‚Üí `{status, output_path}`

Uses `request.app.state.transcriber` (same Whisper model already loaded ‚Äî no reload). GUI and
server are co-located; path-based avoids GB-scale file transfers.

### 5. CLI implementation

Add two optional flags to the existing `generate()` Typer command:
```
--step extract|transcribe|translate|write  (Optional[str], click.Choice)
--work-dir PATH                             (Optional[Path], dir path)
```
Branch executes **before** the existing `run_pipeline()` call ‚Äî full-run mode unchanged.
`--work-dir` is required when `--step` is given; validation raises exit code 2 if absent.

### 6. GUI stepper widget layout

The Generate tab `_frame` grid currently uses rows 0‚Äì11. New rows added at end:
- **Row 12**: "Work directory" section label (CTkLabel, separator style)
- **Row 13**: work-dir entry + Browse button (2-column span)
- **Row 14**: Stepper frame (`CTkFrame`) spanning 3 columns, containing 4 stage sub-frames

Each stage sub-frame (laid out horizontally inside the stepper frame using columns 0‚Äì3):
```
[stage name + status label]
[action button (disabled until prerequisite file exists)]
```

Stage states map to status text + colors:
- `pending`: "‚Äî" (subdued text color)
- `running`: "‚è≥ Running‚Ä¶" (accent color)
- `done`: "‚úì Done" (success green)
- `skipped`: "‚äò Skipped" (warning yellow ‚Äî translate with no target lang)
- `error`: "‚úó Error" (error red)

File-existence polling: `self.after(2000, self._refresh_stepper_state)` on main thread.
Checks `work_dir/{audio.wav, transcription.json, translation.json, subtitles.srt}` to drive
button enable/disable and stage status labels.

**Row 15**: "Clear Work Files" button (`CTkButton`, secondary style)

### 7. `write_srt_step` fallback logic

If `translation.json` exists in `work_dir`, use it as segments input.
If only `transcription.json` exists, use that (no translation case).
This matches D-09: user can skip translate step and write SRT from raw transcription.

## Established Patterns to Follow

From `pipeline.py`:
- Lazy imports inside step functions (avoids import-time side effects)
- `progress_callback: Optional[Callable[[str, int, int], None]]` parameter (optional, no-op default)

From `api/routers/subtitles.py`:
- UploadFile ‚Üí `shutil.copyfileobj()` to temp file ‚Üí process ‚Üí cleanup in `finally`
- Background thread via `threading.Thread(target=..., daemon=True).start()`

From `gui/main.py`:
- Background thread + `self.after(0, callback)` to marshal results to Tk main thread
- `apply_*_btn_style()` / `apply_entry_style()` from `gui/styles.py` for consistent theming

---

## Common Pitfalls to Avoid

- **Do NOT** use `audio_temp_context()` in step mode ‚Äî it deletes the WAV immediately on exit
- **Do NOT** reload WhisperTranscriber in API path endpoints ‚Äî use `request.app.state.transcriber`
- **Do NOT** pass raw `faster_whisper.Segment` objects to JSON ‚Äî they have non-serializable fields
- **Do NOT** modify `run_pipeline()` ‚Äî step functions are additive; `pipeline.py` is read-only
