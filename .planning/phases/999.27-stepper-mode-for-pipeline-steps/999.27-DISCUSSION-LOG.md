## Area 1: Intermediate Artifact Storage

**Q: Where should intermediate step outputs be stored?**
Options: Temp dir (auto-cleaned) / Work dir alongside input / User-specified work dir
**Selected:** User-specified work dir

**Q: Single output dir for all steps, or separate path per step?**
Options: Single output dir for all steps / Intermediates in work dir + final SRT to --output / Per-step output paths
**Selected:** Single output dir for all steps

**Q: When are intermediate files cleaned up?**
Options: Auto-cleaned on new job / Kept until manually cleared / Deleted on full-run completion
**Selected:** Kept until manually cleared

## Area 2: Segments Serialization Format

**Q: How should transcription segments be serialized between steps 2→3 and 3→4?**
Options: JSON (recommended) / Pickle / In-memory (no disk save)
**Selected:** JSON

**Q: What should the JSON file contain?**
Options: Flat JSON array [{start, end, text}] / JSON envelope {language, duration, segments:[...]}
**Selected:** Flat JSON array [{start, end, text}]

**Q: What should the JSON filenames be in the output dir?**
Options: Stage-named files (transcription.json, translation.json) / Step-numbered (step2.json, step3.json) / Agent's discretion

## Area 3: Translate Step When No Target Language

**Q: What's the UX when no target language is configured and user is in step mode?**
Options: Always show + disabled with hint / Hide step if no lang / Always show + mark as Skipped
**Selected:** Always show, mark as Skipped when no lang

**Q: Can user set a target language mid-run (after step 2) and then run Translate retroactively?**
Options: Yes (activate retroactively) / No (must be set upfront)
**Selected:** Yes — user can configure lang and run Translate retroactively
