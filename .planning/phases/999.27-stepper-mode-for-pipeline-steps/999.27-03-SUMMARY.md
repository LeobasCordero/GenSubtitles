## Summary

Added 4 stateless upload/download step endpoints to `gensubtitles/api/routers/subtitles.py`, plus 5 tests in `tests/test_api.py`.

## What Was Built

### gensubtitles/api/routers/subtitles.py
- `POST /subtitles/extract` — video upload → WAV download
- `POST /subtitles/transcribe` — WAV upload → transcription.json download (uses pre-loaded WhisperTranscriber via Depends)
- `POST /subtitles/translate` — segments JSON upload + `target_lang` query param → translation.json download
- `POST /subtitles/write` — segments JSON upload → subtitles.srt download
- All endpoints are stateless (tmp dirs cleaned up via BackgroundTasks)
- Existing `/subtitles/async` and related endpoints unchanged

### tests/test_api.py
- `test_post_subtitles_extract_success` — verifies WAV response, audio/wav content-type
- `test_post_subtitles_transcribe_success` — verifies transcription.json content-disposition
- `test_post_subtitles_translate_success` — verifies translation.json content-disposition
- `test_post_subtitles_write_success` — verifies SRT content with timestamp
- `test_existing_async_endpoint_unchanged` — confirms backward compatibility
