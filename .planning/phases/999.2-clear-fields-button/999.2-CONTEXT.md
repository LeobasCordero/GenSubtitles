# Phase 999.2: GUI — Clear Fields Button - Context

**Gathered:** 2026-04-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a "Clear" button to the GUI that resets the 4 input fields (input video, output SRT, source language, target language) back to empty state. The button is context-aware: it is only enabled when at least one field contains text, and disabled during pipeline execution.

</domain>

<decisions>
## Implementation Decisions

### What "Clear" Resets
- **D-01:** Clear resets **only the 4 input StringVars** (`_input_var`, `_output_var`, `_source_lang_var`, `_target_lang_var`) to `""`. The stage label ("✓ Done", stage names) and the "Open Folder" button (if visible) are **not affected**.

### Reset Values
- **D-05:** All 4 fields reset to `""` — placeholder text reappears naturally (`"auto-detect"`, `"(none — no translation)"`, etc.).

### Button Placement
- **D-02:** Clear button gets **its own row below the Generate button** (Row 5 in the current grid). Progress bar and stage label shift down accordingly.

### Enable/Disable Logic
- **D-03:** Clear button is **only enabled when at least one of the 4 input fields is non-empty**. Starts disabled on app launch. Re-evaluates on every field change via `StringVar.trace_add("write", ...)` on all 4 vars.
- **D-04:** Clear button is **disabled during pipeline execution** (same lifecycle as the Generate button — disabled in `_on_generate`, re-enabled in `_finish_generate`).

### Agent's Discretion
- Button width and visual styling (full-width vs narrower, `fg_color` to distinguish from Generate).
- Exact trace callback implementation (one shared callback vs per-var).
- Whether to use `ctk.CTkButton` `state="disabled"` or `state="normal"` toggling.

</decisions>

### Reusable Assets
- `_input_var`, `_output_var`, `_source_lang_var`, `_target_lang_var` — `ctk.StringVar()` instances; clearing is `.set("")` on each
- `_btn_generate` — uses `.configure(state="disabled"/"normal")` pattern already in `_on_generate`/`_finish_generate`; Clear button follows the same pattern
- `_build_ui()` — grid layout with `self._frame`; Clear button adds as a new row

### Established Patterns
- Button state management: `self._btn_generate.configure(state="disabled")` in `_on_generate`, restored in `_finish_generate` — Clear button must be added to both
- Grid rows: currently Row 4 = Generate, Row 5 = progress bar (hidden), Row 6 = stage label, Row 7 = Open Folder — Clear inserts at Row 5, shifting progress bar to Row 6, stage label to Row 7, Open Folder to Row 8

### Integration Points
- `_on_generate`: add `self._btn_clear.configure(state="disabled")`
- `_finish_generate`: add `self._update_clear_state()` to re-evaluate enabled state
- `_build_ui`: add `self._btn_clear` widget + `StringVar.trace_add` calls for all 4 vars
