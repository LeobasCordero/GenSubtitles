# Phase 999.3: GUI — Auto-populate Subtitle Path from Video - Context

**Gathered:** 2026-04-10
**Status:** Ready for planning

<domain>
## Phase Boundary

When the user selects a video file via the "Browse…" button in the GUI, automatically populate the Output SRT field with the same directory and base filename as the video, replacing the extension with `.srt` (e.g., `/videos/movie.mp4` → `/videos/movie.srt`). Only the `_browse_input` callback is modified — no other triggers.

</domain>

<decisions>
## Implementation Decisions

### Trigger
- **D-01:** Auto-populate fires **only when the user clicks "Browse…"** and selects a file (i.e., inside `_browse_input` after `path` is confirmed non-empty). Typing into the input field manually does **not** trigger auto-populate.

### Overwrite Behavior
- **D-02:** Auto-populate sets the output path **only if `_output_var` is currently empty** (`""`). If the user already has an output path set (manually typed or from a previous browse), it is left untouched.

### Path Derivation
- **D-03:** Output path is derived as `str(Path(video_path).with_suffix(".srt"))`. No special edge-case handling needed — `askopenfilename` always returns an absolute path in browse-only mode, so the formula is unambiguous.

### Agent's Discretion
- Where exactly in `_browse_input` the auto-populate lines are inserted (after `self._input_var.set(path)`, before the method returns).
- Whether to inline the logic or extract a small helper method.

</decisions>

## Existing Code Insights

### Reusable Assets
- `_input_var` / `_output_var` — `ctk.StringVar()` instances; reading with `.get()`, writing with `.set(value)`

### Established Patterns
- `_browse_input` pattern:
  ```python
  def _browse_input(self) -> None:
      from tkinter import filedialog
      path = filedialog.askopenfilename(...)
      if path:
          self._input_var.set(path)
          # auto-populate goes here (after set, inside `if path:`)
  ```
- `_output_var.get()` returns `""` when empty (placeholder text is not the `.get()` value)

### Integration Points
- Only `_browse_input` in `gensubtitles/gui/main.py` needs modification — no other files touched.
