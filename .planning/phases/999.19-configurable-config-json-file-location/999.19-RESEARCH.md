## Architecture Patterns

### Pattern 1: `settings_path()` override

Current implementation:
```python
def settings_path() -> Path:
    from platformdirs import user_config_dir
    return Path(user_config_dir("GenSubtitles")) / "settings.json"
```

Target implementation (D-01, D-04 — env var, fresh read every call):
```python
def settings_path() -> Path:
    import os
    from platformdirs import user_config_dir
    custom = os.environ.get("GENSUBTITLES_CONFIG")
    if custom:
        return Path(custom)
    return Path(user_config_dir("GenSubtitles")) / "settings.json"
```

Priority: `GENSUBTITLES_CONFIG` env var > platformdirs default.

`load_settings()` and `save_settings()` both call `settings_path()` — no changes needed to those functions. The override is transparent.

### Pattern 2: Cross-platform "Open Folder" in `_show_success()` (line 1472)

```python
def _open_folder() -> None:
    sys_name = platform.system()
    if sys_name == "Windows":
        subprocess.Popen(["explorer", str(output_dir)])  # noqa: S603,S607
    elif sys_name == "Darwin":
        subprocess.Popen(["open", str(output_dir)])      # noqa: S603,S607
    else:
        subprocess.Popen(["xdg-open", str(output_dir)]) # noqa: S603,S607
```

Reuse this exact pattern for the config folder button — pass `settings_path().parent` as the directory.

### Pattern 5: `_show_settings()` path refresh

`_show_settings()` (line 1714) populates all fields before showing the panel. The config path label must be refreshed there too:

```python
from gensubtitles.core.settings import settings_path  # noqa: PLC0415
self._lbl_config_path_value.configure(text=str(settings_path()))
```

This ensures the label reflects the *currently active* path each time Settings is opened (important if env var changes between sessions or the panel is opened multiple times in one session).

## Common Pitfalls

1. **Existing tests mock `settings_path` directly** — they will continue to work fine after this change since they replace the function entirely. New tests for env var behavior must NOT mock `settings_path` — they should set `monkeypatch.setenv("GENSUBTITLES_CONFIG", str(path))` and call `settings_path()` directly to test the env var resolution logic.

2. **Save/Back button row shift** — `btn_frame` currently sits at `row=9`. After inserting the config path row at `row=9`, `btn_frame` must be re-gridded to `row=10`. Missing this causes the buttons to disappear.

3. **`sf.columnconfigure`** — Currently only col=1 has `weight=1`. The new row uses col=0 (label), col=1 (path text, stretches), col=2 (button, fixed). Add `sf.columnconfigure(1, weight=1)` is already set; no changes needed to columnconfigure.

4. **`noqa: S603,S607`** — The `subprocess.Popen` call with a list must carry these suppressions, same as in `_show_success()`.

5. **Long paths on Windows** — `settings_path()` may return a long path like `C:\Users\username\AppData\Roaming\GenSubtitles\settings.json`. The label should use `text_color=_p("text_secondary")` and a smaller font to avoid layout overflow. Agent's discretion on truncation.
