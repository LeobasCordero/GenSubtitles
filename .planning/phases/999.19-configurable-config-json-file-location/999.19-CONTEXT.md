# Phase 999.19: Config — Configurable JSON Config File Location - Context

**Gathered:** 2026-04-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Allow users to override the path where `settings.json` is read from and written to. The current hardcoded path (`platformdirs.user_config_dir("GenSubtitles") / "settings.json"`) is made overridable via an environment variable. The GUI Settings panel shows the currently active config path and lets users open the containing folder in the system file explorer.

Out of scope: CLI `--config` flag, portable mode (auto-detect `gensubtitles.json` next to `main.py`).

</domain>

<decisions>
## Implementation Decisions

### Override Mechanism
- **D-01:** Env var only — `GENSUBTITLES_CONFIG`. No CLI flag this phase. The env var is checked in `settings_path()` in `core/settings.py`. This covers all entry points (CLI, API, GUI) uniformly without touching argument parsing.
  - Priority chain: `GENSUBTITLES_CONFIG` env var > default `platformdirs` path
  - If `GENSUBTITLES_CONFIG` is set but the file does not yet exist, `load_settings()` returns defaults and `save_settings()` creates it at that path (same behaviour as the current default path flow).

### Portable Mode
- **D-02:** Deferred. Auto-detection of a `gensubtitles.json` next to `main.py` is NOT part of this phase.

### GUI Settings Path Display
- **D-03:** Add a read-only label showing the resolved config file path in the Settings panel. Alongside it, an "Open folder" button that reveals the config directory in the system file explorer (same pattern as the existing post-generation "Open folder" button — uses `subprocess` / `os.startfile` / `xdg-open`).

### `settings_path()` Implementation
- **D-04:** Read `GENSUBTITLES_CONFIG` from `os.environ` fresh on every call — no module-level caching. Simple to implement and test; env var is set before process start and never changes mid-run.

### Agent's Discretion
- Exact placement of the config path label/button within the Settings panel UI (row position, label text).
- Whether to truncate a very long path with ellipsis or let it wrap.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `settings_path()` in `core/settings.py` (line ~52) — single change point; both `load_settings()` and `save_settings()` call it. Override goes here.
- Existing "Open folder" button in GUI — same cross-platform open-directory pattern (`subprocess.Popen` / `os.startfile`) can be reused for the Settings panel button.

### Established Patterns
- `load_settings()` silently falls back to defaults on missing/corrupt file — keep this behaviour for the env-var-specified path too.
- `save_settings()` calls `path.parent.mkdir(parents=True, exist_ok=True)` before writing — this handles new directories automatically.
- GUI lazy-imports `from gensubtitles.core.settings import ...` inside methods — keep the same pattern for any new settings calls.

### Integration Points
- `settings_path()` → add `os.environ.get("GENSUBTITLES_CONFIG")` check before the `platformdirs` call.
- Settings panel in `gui/main.py` → add a read-only `CTkLabel` (or `CTkEntry` with `state="readonly"`) + `CTkButton` ("Open folder") after the existing settings fields.

</code_context>

<specifics>
## Specific Ideas

- The env var name is `GENSUBTITLES_CONFIG` (matches the app name pattern).
- "Open folder" button in the Settings panel should open the *directory*, not the file itself — consistent with the post-generation button.

</specifics>

<deferred>
## Deferred Ideas

- **CLI `--config` flag** — Deferred. Would need to be a global app-level flag on the Typer `app` and threaded through to `settings_path()`. Revisit in a future phase.
- **Portable mode** — Deferred. Auto-detection of `gensubtitles.json` next to `main.py` for zero-config portable packaging.

</deferred>
