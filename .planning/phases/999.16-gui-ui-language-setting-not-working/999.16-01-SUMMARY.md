# Summary: Fix UI Language Setting

## What was built

Full UI language switching support for English/Spanish in `gensubtitles/gui/main.py`.

## Key changes

- **`_STRINGS` dict** — module-level registry with `"en"` and `"es"` locales, 31 keys each, covering all static labels, buttons, menus, and dialog strings
- **`_s(key)` instance method** — locale lookup using `self._current_settings.ui_language`, falls back to English
- **11 anonymous CTkLabel widgets converted to stored refs**: `_lbl_input_video`, `_lbl_output_file`, `_lbl_target_lang`, `_lbl_output_format`, `_lbl_tl_input_sub`, `_lbl_tl_output_path`, `_lbl_tl_source_lang`, `_lbl_tl_target_lang`, `_lbl_appearance_mode`, `_lbl_ui_language_setting`, `_lbl_default_outdir`
- **`self._settings_menu` and `self._help_menu`** stored in `_build_menu_bar`
- **`_apply_ui_language()`** — relabels all widgets, menus (via `entryconfigure`), and lazy-created buttons. Menu bar uses indices 1/2 (tearoff offset)
- **Wired in `__init__`** after `_apply_startup_target_lang()` for startup language application
- **Wired in `_save_settings`** after `_apply_theme()` for live switch with no restart
- **Dialogs updated**: `_show_tutorial`, `_show_language_pairs`, `_show_about` all use `_s()` for titles and button text

## Deviations

- Fixed a bug discovered during verification: `_menubar` is created without `tearoff=0`, so its cascade entries start at index 1 (not 0). Updated `entryconfigure` calls to use indices 1 and 2 for the menubar.
