**Gathered:** 2026-04-13

<domain>
## Phase Boundary

Fix duplicate entries appearing in the "Installed Language Pairs" Help menu dialog and
ensure `GET /languages` API response is also duplicate-free. Deduplication applied at the
source function so all callers benefit automatically.

</domain>

<decisions>
## Implementation Decisions

### Fix Location
- **D-01:** Apply deduplication in `list_installed_pairs()` in `gensubtitles/core/translator.py` only — single fix, one place to maintain, all callers (API, GUI) benefit automatically via the call chain.
- **D-02:** Do NOT add defensive dedup layers in `GET /languages` or the GUI dialog — the source fix is sufficient.

### Test Coverage
- **D-03:** Add a unit test in `tests/test_translator.py` that mocks Argos Translate returning duplicate language objects and asserts `list_installed_pairs()` returns each `from→to` pair exactly once.

### the agent's Discretion
- Dedup implementation approach (set comprehension vs seen-set loop) — agent chooses idiomatic Python.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `list_installed_pairs()` current implementation (translator.py:57): iterates `get_installed_languages()` and appends `{"from": lang.code, "to": t.to_lang.code}` — no dedup guard present.

### Established Patterns
- Existing tests in `test_translator.py` mock `argostranslate.translate` — same pattern to use for the new test.

### Integration Points
- `GET /languages` (subtitles.py:165): `return {"pairs": list_installed_pairs()}` — no intermediate processing; source fix propagates here cleanly.
- `_populate_language_dropdowns()` (gui/main.py:1027): sets `self._language_pairs = pairs` from API response — also cleaned automatically.

### Root Cause
Argos Translate may register the same language package multiple times after reinstall or index refresh, causing `get_installed_languages()` to yield duplicate language objects with identical `code` values. The nested loop then emits the same `(from, to)` pair multiple times.

</code_context>

<specifics>
## Specific Ideas

- ROADMAP suggests: `{(p["from"], p["to"]) for p in pairs}` set comprehension approach — acceptable but loses dict structure; implementation may use an ordered seen-set for a cleaner list result.
- No specific UI changes needed — the deduplicated data automatically renders correctly in `_show_language_pairs()`.

</specifics>
