# 999.30-03 SUMMARY — Build 4 step tabs with per-tab execution and pre-fill chain

**Status:** COMPLETE
**Commit:** 6aa82c6
**Files modified:** `gensubtitles/gui/main.py`

## What was done

### Task 1 — Build step tab methods

Added 4 new build methods to `GenSubtitlesApp`:

- `_build_extract_tab()` — Tab 3: input video entry + browse, output audio entry + save-as, Extract button (accent, disabled until server ready), Clear button (secondary), log textbox (`_extract_log_textbox`)
- `_build_transcribe_tab()` — Tab 4: input audio entry + browse, source language dropdown (shares `_source_lang_var`), Transcribe button, Clear button, log textbox (`_transcribe_log_textbox`)
- `_build_translate_step_tab()` — Tab 5: input transcription entry + browse, source/target lang dropdowns, engine dropdown (hidden until target selected), Translate button, Clear button, log textbox (`_translate_step_log_textbox`)
- `_build_write_tab()` — Tab 6: input transcription entry + browse, output path entry + save-as, format dropdown (SRT/SSA), Write button, Clear button, log textbox (`_write_log_textbox`)

All 4 called from `_build_ui()` after `_build_translate_tab()`.

Added helper methods:
- `_browse_file_to_var(var, filetypes)` — opens file-open dialog, sets StringVar
- `_save_file_to_var(var, defaultextension, filetypes)` — opens save dialog, sets StringVar
- `_clear_tab_vars(*vars)` — clears one or more StringVars
- `_on_translate_step_target_change(value)` — shows/hides engine dropdown in Tab 5

### Task 2 — Rewrite step callbacks + integrate

Rewrote `_run_step_in_bg` to accept `log_textbox`, `on_success`, `on_error` kwargs — logs route to the correct tab's textbox.

Replaced old `_on_step_extract/transcribe/translate/write` with per-tab handlers:
- `_on_tab3_extract()` → calls `/steps/extract`, pre-fills `_transcribe_input_var` on success (D-04)
- `_on_tab4_transcribe()` → calls `/steps/transcribe`, pre-fills `_translate_step_input_var` on success (D-04)
- `_on_tab5_translate()` → calls `/steps/translate`, pre-fills `_write_input_var` on success (D-04)
- `_on_tab6_write()` → calls `/steps/write`, logs completion

`_on_step_success` and `_on_step_error` kept as legacy no-ops for compatibility.

Extended `_on_server_ready()` to enable all 4 step tab action buttons after server is ready.

Extended `_apply_ui_language()` with hasattr-guarded blocks to configure all step tab labels and buttons in both en and es.

## Key decisions

- Pre-fill chain (D-04): extract → transcribe → translate → write, using glob for .wav detection and TRANSCRIPTION_FILENAME / TRANSLATION_FILENAME constants
- `_source_lang_var` shared between Tab 1 (generate pipeline) and Tab 4 (transcribe) — consistent language selection
- Engine dropdown in Tab 5 hidden until target language selected (matches Tab 1 behavior)
- All run buttons start `state="disabled"`, enabled by `_on_server_ready()`
