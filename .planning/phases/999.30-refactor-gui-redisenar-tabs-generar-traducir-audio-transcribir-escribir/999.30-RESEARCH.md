### Log helper universal
```python
def _log_to(self, textbox: ctk.CTkTextbox, msg: str) -> None:
    try:
        textbox.configure(state="normal")
        textbox.insert("end", msg + "\n")
        textbox.configure(state="disabled")
        textbox.see("end")
    except Exception:
        pass
```
`_log(msg)` se convierte en alias: `self._log_to(self._log_textbox, msg)` para Tab 1.

### Work dir por tab
```python
def _get_tab_work_dir(
    self,
    input_var: ctk.StringVar,
    fallback_vars: list[ctk.StringVar] | None = None
) -> "Path | None":
    val = input_var.get().strip()
    if val:
        return Path(val).parent
    if fallback_vars:
        for fv in fallback_vars:
            fv_val = fv.get().strip()
            if fv_val:
                return Path(fv_val).parent
    return None
```
Fallback chain (D-05): tab propio → `_input_var` (Tab 1 video) → `_extract_input_var` (Tab 3 audio).

## Pitfalls Identificados

1. **`_run_step_in_bg` logging**: actualmente llama a `self._log(msg)` que escribe en `_log_textbox` (Tab 1). Para tabs 3-6, hay que pasar el textbox correcto. Solución: agregar `log_textbox` param al método.

2. **`_on_step_success` / `_on_step_error`**: actualmente actualizan `_step_status_labels[step_key]` y `_stage_label`. Estos widgets del stepper desaparecen. Hay que refactorizar para no depender de widgets del stepper.

3. **`_apply_theme()` referencias**: actualmente itera sobre widget refs explícitas. Nuevos log textboxes de tabs 3-6 necesitan ser incluidos para que `fg_color=p("input_bg")` se aplique en tema-cambio.

4. **`_relabel_tabview_headers()` bidireccionalidad**: el `tab_name_map` necesita incluir TANTO el nombre inglés base COMO el nombre localizado como "fuente" (para que re-aplicar funcione correctamente). Requiere agregar claves para los 4 nuevos tabs en ambas lenguas.

5. **`_on_server_ready()`**: habilita `_btn_generate` y `_tl_btn_translate`. Los botones Acción de tabs 3-6 también deben habilitarse aquí (o condicionalmente en trace de sus input vars).

6. **Tab 5 vs Tab 2 nombre collision**: "Translate" y "Translate Subtitles" — usar nombres distintos. Tab 5 usa `translate_step_tab` key para evitar colisión.

7. **Stepper vars en `__init__`**: `_step_states`, `_step_buttons`, `_step_status_labels`, `_stepper_refresh_id`, etc. deben eliminarse del `__init__` junto con la cancelación del `after()` timer en `on_closing`.

8. **Tab 4 output**: el endpoint `/steps/transcribe` no acepta `output_path` — escribe siempre a `work_dir/transcription.json`. El campo "output" en Tab 4 es informativo (muestra el path que se generará) o se omite.
