# Phase 999.29 — Context: GUI Rediseño Layout Tres Paneles Verticales

## Phase Goal

Reorganizar el layout del tab "Generate Subtitles" de una sola columna (grid 3-col: label | entry | button) a **tres paneles `CTkFrame` side-by-side** con responsabilidades separadas, sin perder ninguna funcionalidad existente.

---

## Canonical Refs

- `gensubtitles/gui/main.py` — Archivo principal de la GUI. Contiene `GenSubtitlesApp`, `_build_ui()`, `_build_stepper_section()` — el objetivo de esta fase.
- `gensubtitles/gui/theme.py` — Paleta de colores dual Dark/Light. Tokens: `p("bg")`, `p("surface")`, `p("accent")`, `p("secondary")`.
- `gensubtitles/gui/styles.py` — Helpers de estilo: `apply_accent_btn_style()`, `apply_secondary_btn_style()`, `apply_entry_style()`, `apply_progress_bar_style()`, `apply_stage_label_style()`.
- `gensubtitles/gui/locale.py` — Sistema de localización `s()` / `set_language()` — todos los strings visibles al usuario deben pasar por aquí.
- `.planning/ROADMAP.md` — Descripción completa de la fase (sección Phase 999.29).

---

## Decisions

### 1. Scope de la migración

- **Reemplaza `_build_ui()` in-place** para el tab "Generate Subtitles" — refactor completo, mismo objeto `CTkTabview`, mismos `StringVar`s y callbacks existentes.
- El tab "Translate Subtitles", el panel de Settings y el menú bar quedan **exactamente intactos** — cero cambios fuera del tab Generate.
- Todos los `StringVar`s existentes (`_input_var`, `_output_var`, `_source_lang_var`, `_target_lang_var`, `_output_format_var`, `_engine_var`, `_work_dir_var`) se mantienen — son la fuente de verdad del estado de la app.

### 2. Estructura del nuevo layout — Tres paneles iguales (1:1:1)

Tres `CTkFrame` side-by-side con **columnconfigure weight=1 en los tres**, dentro del frame del tab "Generate Subtitles".

```
+------------------+------------------+------------------+
|   files_frame    |  process_frame   |   status_frame   |
|  "Pasos iniciales"| "Configuración"  | "Control y Progreso"|
+------------------+------------------+------------------+
```

- Cada panel: `CTkFrame` con `corner_radius=12`, fondo `p("surface")` (ligeramente diferente al `p("bg")` del fondo principal).
- El título de cada panel: `CTkLabel` con estilo `font("subheader")`, alineado al top.

### 3. Panel Izquierdo — `files_frame` ("Pasos iniciales")

Contenido:
- CTkLabel título: "Pasos iniciales" 
- Grupo "Video de entrada": `CTkEntry` (textvariable=`_input_var`) + `CTkButton` "Explorar…" (estilo secundario `apply_secondary_btn_style()`)
- Grupo "Archivo de salida": `CTkEntry` (textvariable=`_output_var`) + `CTkButton` "Guardar como…" (estilo secundario)

Callbacks existentes: `_browse_input()`, `_browse_output()` — sin cambios.

### 4. Panel Central — `process_frame` ("Configuración")

Contenido (en orden vertical):
- CTkLabel título: "Configuración"
- CTkOptionMenu "Idioma de origen" (variable=`_source_lang_var`, command=`_on_source_lang_change`)
- CTkOptionMenu "Idioma de destino" (variable=`_target_lang_var`, command=`_on_target_lang_change`)
- CTkOptionMenu "Motor de traducción" (variable=`_engine_var`, valores condicionales Argos/DeepL/LibreTranslate igual que hoy)
- CTkOptionMenu "Formato de salida" (variable=`_output_format_var`, valores SRT/SSA)

Todos los dropdowns se populan dinámicamente igual que en el código actual.

### 5. Panel Derecho — `status_frame` ("Control y Progreso")

Contenido (en orden vertical):

**a) CTkSwitch — "Modo paso a paso"**
- `_stepper_switch_var = ctk.BooleanVar(value=False)`
- Al cambiar: actualiza el texto del botón principal y el estado de los stage-items.

**b) Flujo visual de 4 etapas (horizontal dentro del panel)**
- 4 sub-frames horizontales: `1. Extract` → `2. Transcribe` → `3. Translate` → `4. Write SRT`
- Cada etapa muestra:
  - Label con nombre de etapa
  - Label de estado: `—` / `⏳ Pendiente` / `✅ Listo` / `⚠️ Error` (texto, sin íconos externos)
- Al hacer **click en una etapa** en modo paso a paso: activa la ejecución de ese paso si es el siguiente disponible (wizard semi-interactivo)
- Los stage-items funcionan como `CTkButton` o frame clickeable cuando el paso está disponible; sin efecto cuando no lo está

**c) Botón principal de acción (grande, acento)**
- Estilo `apply_accent_btn_style()`, height grande (igual que el `_btn_generate` actual)
- Texto **dinámico** según el switch:
  - Switch OFF: `"Generar Subtítulos"` (flujo completo, equivale al botón actual)
  - Switch ON: `"Iniciar Proceso Paso a Paso"` → cambia a `"Ejecutar siguiente paso"` tras cada ejecución
- Command: `_on_generate()` (sin cambios en la lógica)

**d) Barra de progreso**
- `CTkProgressBar` con `apply_progress_bar_style()` — igual que hoy
- Se muestra/oculta en los mismos momentos que el código actual

**e) Stage label**
- `CTkLabel` con `apply_stage_label_style()` — coexiste con el log (no se elimina)
- Muestra la etapa actual en tiempo real

**f) CTkTextbox — Registro de actividad (log)**
- Pequeño, deshabilitado para edición por el usuario (`state="disabled"`)
- Fondo `p("input_bg")`, fuente `font("mono")`
- Auto-scroll al final con cada mensaje nuevo
- Recibe los mismos mensajes que actualmente van al stage_label + mensajes de carga del modelo del servidor
- Para escribir: `textbox.configure(state="normal")` → insert → `configure(state="disabled")` → `see("end")`

**g) Botón Cancel**
- Se mantiene igual, visible solo durante ejecución activa

### 6. Tamaño de ventana

- `minsize(1100, 600)` — suficiente espacio para los tres paneles
- `self.configure(fg_color=p("bg"))` — sin cambios

### 7. Diferenciación de colores

- Fondo ventana: `p("bg")` (#121212 dark / #F5F5F5 light) — sin cambios
- Fondo paneles (`files_frame`, `process_frame`, `status_frame`): `p("surface")` (#1E1E1E dark / #FFFFFF light) — ligeramente diferente
- Botones "Explorar…" / "Guardar como…": `apply_secondary_btn_style()` — color secundario
- Botón "Generar Subtítulos": `apply_accent_btn_style()` — color de acento principal

### 8. Work directory picker

- El `_entry_work_dir` + `_btn_browse_work_dir` del stepper se integra en el panel derecho, encima del flujo de 4 etapas (solo visible cuando switch está ON).

### 9. Localización

- Todos los strings nuevos visibles al usuario deben registrarse en `locale.py` mediante el sistema `s()` existente — no strings hardcodeados en `main.py`.

---

## Non-Goals (fuera de scope)

- No modificar el tab "Translate Subtitles"
- No modificar el panel de Settings ni el menú bar
- No agregar nuevas funcionalidades de pipeline
- No cambiar callbacks, lógica de negocio ni StringVars existentes

## Deferred Ideas

- Animaciones de transición entre etapas del stepper (fuera de scope — sería Phase 999.30+)
- Responsividad / colapso de paneles en ventanas pequeñas (fuera de scope)
