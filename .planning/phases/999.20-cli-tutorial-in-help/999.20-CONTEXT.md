# Phase 999.20: Docs — CLI Tutorial — Context

**Gathered:** 2026-04-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Write a dedicated, standalone CLI tutorial (`docs/cli-tutorial.md`) that walks technical users through every CLI command end-to-end with narrative, real command examples, and expected output snippets. Also update the GUI Tutorial dialog (`locale.py`) to cross-reference the CLI tutorial, and update README to link to the new doc.

This phase does NOT cover GUI usage content — that already lives in the GUI Tutorial dialog. It does NOT add new CLI features.

</domain>

<decisions>
## Implementation Decisions

### Document Location
- **D-01:** Standalone `docs/cli-tutorial.md` — NOT a new section in README.md. README stays as a quick reference; the tutorial is the long-form narrative. README's existing "CLI Usage" section gets a one-line link to the new doc.

### Content Scope
- **D-02:** Full coverage — all commands get dedicated step-by-step sections:
  - `generate` (default command) — basic transcription, with-translation, format options, engine selection
  - `translate` — translating an existing subtitle file
  - `convert` — SRT ↔ SSA conversion
  - `serve` — starting the API server, with curl example hitting `POST /subtitles`
  - `--step` stepper mode — all four stages: extract, transcribe, translate, write; including work-dir usage
  - `--engine` flag — argos vs deepl vs libretranslate, when to use each
  - `--format` flag — SRT vs SSA trade-offs

### Model Timing Data
- **D-03:** Qualitative descriptions only — describe models as "fastest / balanced / most accurate" with download sizes (e.g., tiny ~75 MB, small ~470 MB, medium ~1.5 GB, large ~3 GB). No numeric timing benchmarks. Hardware varies too much to be meaningful without a disclaimer-heavy table.

### GUI Help Menu Cross-link
- **D-04:** Two-way cross-link:
  - `docs/cli-tutorial.md` footer: note pointing to `python main.py gui` for the GUI
  - GUI Tutorial dialog (`gensubtitles/gui/locale.py`) EN `dlg_tutorial_text`: add a line at the bottom pointing to `docs/cli-tutorial.md` for CLI users
  - GUI Tutorial dialog ES `dlg_tutorial_text`: same addition in Spanish
  - Both EN and ES strings must be updated

### Agent's Discretion
- Tutorial structure: ordered sections from simplest to most advanced (generate → translate → convert → serve → --step stepper → flags reference)
- Expected output snippets: show the `[1/4] Extracting audio...` style progress lines and the final `Done: output.srt (42 segments, lang=en)` line — these are the actual CLI output format
- Installation prereqs section at the top (Python, FFmpeg, pip install) — brief, since README covers this in depth; just enough context for the tutorial to be self-contained

</decisions>
