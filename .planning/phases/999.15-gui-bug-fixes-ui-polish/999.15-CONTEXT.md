# Phase 999.15: GUI ‚Äî UI Bug Fixes & Polish ‚Äî Context

**Gathered:** 2026-04-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix 5 discrete GUI bugs in `gensubtitles/gui/main.py`. No changes to backend, CLI, API, or other
modules except one additive field to `AppSettings` in `gensubtitles/core/settings.py` (BUG-3).

All 5 bugs are **self-contained and independent** ‚Äî they can be planned/executed as separate tasks
or combined into one plan.

**Bugs in scope:**
- BUG-1: "Open Folder" button persists when "Generate Subtitles" is clicked a second time
- BUG-2: DeepL and LibreTranslate visible in the engine dropdown even when not configured
- BUG-3: Target language dropdown does not initialize from `AppSettings` on startup
- BUG-4: About dialog's Close and GitHub buttons don't follow app color/font conventions
- BUG-5: In light theme, Generate and Clear buttons are nearly invisible when disabled

**New git branch created:** `feature/999.15-gui-bug-fixes-ui-polish`
</domain>

<decisions>
## Implementation Decisions

### BUG-1: Open Folder button hidden on re-run
- **D-01:** At the very start of `_on_generate` (before any validation), call
  `self._btn_open_folder.grid_remove()` if the attribute exists (`hasattr` guard required ‚Äî the
  button is created lazily in `_show_success` on first success).
- **D-02:** No change to `_finish_generate` or `_show_success` ‚Äî existing logic already shows the
  button correctly on success.

### BUG-2: Engine dropdown ‚Äî hide unconfigured engines
- **D-03:** Engine dropdown shows ONLY `"Argos"` until DeepL or LibreTranslate is configured.
  Determination: read `_current_settings` at the time the engine list is built.
  - If `_current_settings.deepl_api_key` is non-empty ‚Üí include `"DeepL"` in values.
  - If `_current_settings.libretranslate_url` is non-empty ‚Üí include `"LibreTranslate"` in values.
- **D-04:** Engine list is rebuilt in `_build_ui` (initial construction) AND in `_save_settings`
  (after the user saves new credentials ‚Äî the dropdown must update immediately without a restart).
  Use `self._option_engine.configure(values=...)` in `_save_settings` to refresh the list.
- **D-05:** If the currently selected engine is removed from the list (e.g., user deletes their
  DeepL key), reset `_engine_var` to `"Argos"`.
- **D-06:** The engine row (label + dropdown) remains hidden when "No target" is selected, as
  before. D-03 only affects the VALUES in the dropdown, not its visibility logic.

### BUG-3: Target language dropdown initialized from AppSettings
- **D-07:** Add `target_lang: str = ""` field to `AppSettings` dataclass in
  `gensubtitles/core/settings.py`. Empty string = "No target" (no translation).
- **D-08:** In `_apply_startup_settings`, after loading settings, store `target_lang` internally.
  After `_build_ui` completes (called from `__init__` between `_apply_startup_settings` and
  `_apply_startup_theme`), apply the saved target language to `_target_lang_var` and trigger
  `_on_target_lang_change` to show/hide the engine row. Best location: a new small helper called
  from `_apply_startup_theme` or directly in `__init__` after `_build_ui`.
- **D-09:** Auto-save `target_lang` on every change: in `_on_target_lang_change`, after updating
  UI, call `save_settings` with the updated `target_lang` value. Use a try/except to swallow
  persistence errors silently (non-critical).
- **D-10:** Display-name to ISO-code mapping already exists via `_label_to_code()`. Store as ISO
  code in settings; on startup, map ISO code back to display label using `_CODE_TO_LABEL` before
  setting `_target_lang_var`.

### BUG-4: About dialog styling
- **D-11:** Replace `fg_color="gray"` on the Close button with
  `fg_color=_p("secondary"), hover_color=_p("secondary_hov")`.
- **D-12:** Apply `fg_color=_p("accent"), hover_color=_p("accent_hov")` to the GitHub Project
  button (currently uses CTkButton defaults).
- **D-13:** Add `fg_color=_p("bg")` to the `CTkToplevel` window itself so it follows the palette
  rather than CTk's default surface color.
- **D-14:** No structural/layout changes ‚Äî spacing, label text, and font calls are already correct
  (`_p("text_primary")` and `_font()` used on all labels). Only button colors need fixing.

### BUG-5: Disabled button appearance in light theme
- **D-15:** Scope limited to `_btn_generate` and `_btn_clear` only ‚Äî these are the two most
  visible offenders.
- **D-16:** Set `text_color_disabled` explicitly on both buttons at construction time:
  - `_btn_generate`: `text_color_disabled=("#757575", "#9E9E9E")` ‚Äî tuple (light, dark). `#757575`
    on light `#1976D2` background passes WCAG AA for large text.
  - `_btn_clear`: `text_color_disabled=("#757575", "#9E9E9E")` ‚Äî same tuple. `#757575` on light
    `#E0E0E0` background is 3.7:1 ratio ‚Äî passes WCAG AA large text.
- **D-17:** No changes to `_apply_theme` ‚Äî `text_color_disabled` is set at construction and
  persists across theme switches automatically (CustomTkinter handles it).
</decisions>

<existing_patterns>
## Codebase Patterns to Follow

- **Color tokens:** Always `_p("token")` ‚Äî never hardcoded hex in widget constructors
- **Font:** Always `_font("role")` ‚Äî never `ctk.CTkFont(...)` inline
- **Visibility:** `.grid_remove()` / `.grid()` for show/hide ‚Äî widgets stay in memory
- **Disable cycle:** `widget.configure(state="disabled")` in `_on_generate`,
  `state="normal"` in `_finish_generate` ‚Äî do NOT change this cycle for BUG-5
- **Lazy widget guard:** `hasattr(self, "_btn_open_folder")` before accessing lazily-created widgets
- **Settings persistence:** `load_settings()` / `save_settings()` from `gensubtitles.core.settings`
  imported inside method bodies (not at module level ‚Äî headless env compatibility)
</existing_patterns>
