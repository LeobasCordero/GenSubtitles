# Phase 999.28: Stepper Work-Dir Auto-Subfolder and Named Audio Output - Context

**Gathered:** 2026-04-16
**Status:** Ready for planning

<domain>
## Phase Boundary

Improve stepper mode artifact isolation with two related changes:

1. **Auto-subfolder:** When the GUI user picks a parent folder via Browse, the actual
   `work_dir` used at step execution time is `<parent>/<sanitized-video-stem>/` — so
   multiple videos can share the same parent folder without overwriting each other's
   artifacts.

2. **Named audio:** The extracted audio file is named after the video stem
   (e.g. `myvideo.wav`) instead of the fixed `audio.wav` constant. This applies to
   `core/steps.py` and flows through all surfaces (GUI, CLI, API).

These two changes together eliminate per-video artifact collision when a shared parent
folder is reused across multiple jobs.

</domain>

<decisions>
## Implementation Decisions

### Scope
- **D-01:** Both changes are in scope and apply to all surfaces (GUI, CLI, FastAPI).
  `core/steps.py` is updated — not a GUI-only workaround.

### Audio File Discovery in `transcribe_step()`
- **D-02:** `transcribe_step()` (and any step that reads the audio file) discovers the
  WAV by globbing `*.wav` in `work_dir`. If 0 or >1 WAV files are found, the step
  raises a clear error. No hardcoded `audio.wav` constant for discovery.

### Auto-Subfolder Timing (GUI)
- **D-03:** The subfolder is computed and created **at step execution time** (when the
  user clicks "Extract Audio"), not when the Browse dialog closes. The work-dir entry
  field continues to show the parent path the user selected — the derived
  `parent/stem/` path is passed internally to the API call without updating the
  displayed field.

### Stem Sanitization
- **D-04:** The video filename stem is sanitized before use as a subfolder/audio name:
  spaces replaced with underscores, special characters stripped (keep alphanumeric,
  hyphens, underscores). Example: `"My Video (2024).mp4"` → `My_Video_2024`.

### Browse vs Manual Path (GUI)
- **D-05:** Auto-subfolder logic applies **only when the work_dir was set via the Browse
  button**. If the user typed a path manually into the entry field, that path is used
  as-is — no stem is appended. This allows power users to point directly at an existing
  artifact directory without interference.
  Implementation note: track a boolean `_work_dir_from_browse: bool` flag on the widget;
  set `True` on browse, reset to `False` when the entry is edited manually.

### Agent's Discretion
- Whether `extract_audio_step()` accepts an explicit `audio_filename` parameter or
  derives the name from `video_path` stem internally — choose whichever keeps the call
  sites simplest.
- Whether CLI's `--work-dir` auto-appends the video stem automatically or requires the
  user to specify the full subfolder path — keep CLI behavior minimal and explicit.
- Error message wording when 0 or >1 WAV files are found in `transcribe_step()`.

</decisions>
