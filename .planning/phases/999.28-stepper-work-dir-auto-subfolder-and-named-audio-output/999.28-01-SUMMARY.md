# Plan 999.28-01 Summary

## What Was Built

- **`sanitize_stem(name)`** helper added to `gensubtitles/core/steps.py` — replaces spaces with underscores, strips non-alphanumeric/hyphen/underscore characters (e.g. `"My Video (2024)"` → `"My_Video_2024"`)
- **`extract_audio_step()`** updated to write `{sanitize_stem(stem)}.wav` instead of `audio.wav` — no filename collisions when multiple videos share a work dir
- **`transcribe_step()`** updated to discover the WAV via `work_dir.glob("*.wav")` — raises `FileNotFoundError` with a clear message on 0 or >1 matches
- **`/subtitles/extract` API endpoint** fixed to use the returned `wav_path` from `extract_audio_step()` instead of hardcoded `audio.wav`
- **`AUDIO_FILENAME` constant** kept for backwards-compat (imported by `gui/main.py` and `api/routers/steps.py`)

## Key Files

- `gensubtitles/core/steps.py` — `sanitize_stem`, updated `extract_audio_step`, updated `transcribe_step`
- `gensubtitles/api/routers/subtitles.py` — `/subtitles/extract` fixed to use returned wav_path
- `tests/test_steps.py` — 4 new `sanitize_stem` tests, updated `extract_audio_step`/`transcribe_step` tests, new multiple-WAV test
- `tests/test_api_steps.py` — updated `test_steps_extract_success` assertion to match stem-named WAV
