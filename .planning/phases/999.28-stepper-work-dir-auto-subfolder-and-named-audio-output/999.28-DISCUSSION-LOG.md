## Area A — Scope of Changes

**Question:** Should the named audio output apply to GUI only, or to all surfaces (CLI + API)?

| Option | Description |
|--------|-------------|
| GUI only | CLI and API keep `audio.wav`; only GUI changes |
| Named audio everywhere | CLI/API also get `<stem>.wav`; requires changing `extract_audio_step()` signature |
| Auto-subfolder in GUI + named audio in core | Both changes, all surfaces |

**Selected:** Auto-subfolder in GUI + named audio in core (option 3)

## Area B — Audio File Discovery in `transcribe_step()`

**Question:** How should `transcribe_step()` locate the audio file when it's no longer `audio.wav`?

| Option | Description |
|--------|-------------|
| Pass `audio_filename` explicitly | Caller passes stem through; explicit, no magic |
| Glob for `*.wav` in work_dir | Scan for any WAV; error if 0 or >1 |
