## Summary

Overhauled the core translation engine with context-batching and new engine backends.

## What Was Built

**Task 1: Argos context batching**
- Replaced per-segment loop in `translate_segments()` with XML-marker batch approach
- Each hop now sends all segments as one string: `<1>text</1><2>text</2>`
- `re.findall` parses markers back to per-segment texts after translation
- Count mismatch raises `RuntimeError("Batched translation marker mismatch: expected N segments, got M")`
- Progress callback updated to per-hop semantics: `progress_callback(hop_idx + 1, len(route))`

**Task 2: Engine dispatch + credentials**
- Added `engine: str = "argos"` parameter to `translate_segments()`
- Added `_translate_deepl()`: checks `deepl_api_key` before importing `deepl`; calls `deepl.Translator(key).translate_text(texts, target_lang=LANG.upper())`
- Added `_translate_libretranslate()`: calls `requests.post(url/translate, json=...)` per segment
- Added 3 fields to `AppSettings`: `deepl_api_key`, `libretranslate_url`, `libretranslate_api_key`
- Added `deepl>=1.18` to `pyproject.toml` and `requirements.txt`
- Added `from gensubtitles.core.settings import load_settings` at module level for patchability
