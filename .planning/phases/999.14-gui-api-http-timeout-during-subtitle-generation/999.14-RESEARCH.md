| Component | What to use | Why |
|-----------|-------------|-----|
| FastAPI SSE | `StreamingResponse(generator, media_type="text/event-stream")` | Built-in to Starlette — no extra dep |
| SSE client | `requests` `stream=True` + `iter_lines(decode_unicode=True)` | Already in deps; matches existing GUI pattern |

## Architecture Patterns

### 3-Step Flow (D-01)

```
GUI thread                    API (FastAPI)                  Pipeline (sync thread)
─────────────────────────────────────────────────────────────────────────────────
POST /subtitles/async ──────► create job, start thread ──► start pipeline
◄── {"job_id": "<uuid4>"} ──┘                              │
GET /subtitles/{id}/stream ─► StreamingResponse ◄──────── q.put({stage, ...})
◄── data: {...}\n\n ────────  (async generator reads         │
     ... events ...           from queue via executor)        │
◄── data: {"stage":"done"}  ◄───────────────────────────── q.put({"stage":"done"})
GET /subtitles/{id}/result ─► FileResponse(result_path)
```

### Job Store Shape

```python
_jobs: dict[str, dict] = {}

# Each entry:
{
    "queue":  queue.Queue(),           # progress events
    "cancel": threading.Event(),       # set → pipeline stops between stages
    "result": Path | None,             # set when pipeline writes SRT
    "error":  str | None,             # set on pipeline failure
    "done":   threading.Event(),       # set when queue stream is closed
}
```

### SSE Endpoint Pattern (FastAPI)

```python
from fastapi.responses import StreamingResponse
import asyncio, json, queue

@router.get("/subtitles/{job_id}/stream")
async def stream_job(job_id: str):
    if job_id not in _jobs:
        raise HTTPException(404)
    job = _jobs[job_id]
    q: queue.Queue = job["queue"]
    loop = asyncio.get_running_loop()

    async def _generator():
        while True:
            event = await loop.run_in_executor(None, q.get)  # blocking get, offloaded
            yield f"data: {json.dumps(event)}\n\n"
            if event.get("stage") in ("done", "error", "cancelled"):
                break

    return StreamingResponse(_generator(), media_type="text/event-stream")
```

### Pipeline → Queue Integration

The existing `_set_progress()` helper is extended to also enqueue events:

```python
def _set_progress(stage, label, current=0, total=0, job=None):
    with _progress_lock:
        _progress["stage"] = stage
        ...
    if job is not None:
        job["queue"].put({"stage": stage, "label": label, "current": current, "total": total})
```

Each pipeline stage call passes the `job` dict. On `done` or `error`, puts the final event and closes.

### Cancellation Flag Check Points (D-07)

The pipeline checks `job["cancel"].is_set()` at three points:
1. After `extract_audio()` completes
2. After `transcriber.transcribe()` completes
3. After `translate_segments()` completes (or if skipped, after transcription)

On detection: clean temp files, `q.put({"stage": "cancelled", "label": "Cancelled"})`, return.

### SSE Client in GUI (requests)

```python
import requests as req, json as _json

def _run_sse_stream(self, job_id, input_path, output_path):
    with req.get(
        f"{_BASE_URL}/subtitles/{job_id}/stream",
        stream=True,
        timeout=None,  # no timeout — stream stays open until done
    ) as resp:
        for line in resp.iter_lines(decode_unicode=True):
            if not line or not line.startswith("data:"):
                continue
            data = _json.loads(line[5:].strip())
            if not self._closing:
                self.after(0, lambda d=data: self._apply_progress(d))
            stage = data.get("stage")
            if stage == "done":
                # GET result
                ...
                self.after(0, self._finish_generate, None, final_path)
                return
            if stage in ("error", "cancelled"):
                msg = data.get("label", "Unknown error")
                self.after(0, self._finish_generate, msg, None)
                return
```

`timeout=None` is critical — the SSE stream must stay open indefinitely. This replaces the `timeout=3600` that caused the original bug.

### DELETE /subtitles/{job_id} Cancel Endpoint

```python
@router.delete("/subtitles/{job_id}")
def cancel_job(job_id: str) -> dict:
    job = _jobs.get(job_id)
    if not job:
        raise HTTPException(404, "Job not found")
    job["cancel"].set()
    return {"status": "cancelling"}
```

### Cancel Button (D-06)

- Widget: `ctk.CTkButton` in generate tab progress area, placed in a new row below `_stage_label`
- State: hidden until job starts (`grid_remove()`), shown when `_job_active = True`
- Palette: `fg_color=_p("progress_err")` (red), `hover_color` slightly lighter
- On click: POST `DELETE /subtitles/{self._current_job_id}` in background thread, disable button immediately

## Don't Hand-Roll

- **Do NOT** use `sse-starlette` — it adds a dep for zero gain; `StreamingResponse` is sufficient
- **Do NOT** use `asyncio.Queue` for the sync→async bridge — passing loop references to sync pipeline threads is fragile; `queue.Queue` + `run_in_executor` is cleaner
- **Do NOT** keep `GET /progress` alive alongside SSE — D-02 retires it
- **Do NOT** use `timeout=(connect_timeout, read_timeout)` tuple for the SSE stream — `None` is correct

---

## Common Pitfalls

1. **`iter_lines()` returns bytes on some platforms** — always pass `decode_unicode=True`
2. **Job store cleanup**: After `GET /result` succeeds, delete the job from `_jobs` and unlink temp files. This prevents memory/disk leaks on long sessions.
3. **GUI cancel button still shown after error/cancel**: `_finish_generate` must hide the cancel button in all paths (error, success, cancel)
4. **SSE stream left open if GUI closes**: The GUI's `_closing` guard prevents scheduling more UI updates, but the stream thread should also check `self._closing` before the next `q.get` iteration to stop blocking. Since `_finish_generate` is the final step, and threads are daemon threads, this isn't catastrophic — but adding a `self._closing` check before `q.get` is cleaner.
5. **`_current_job_id` attribute**: Add to `__init__` as `self._current_job_id: str | None = None`
