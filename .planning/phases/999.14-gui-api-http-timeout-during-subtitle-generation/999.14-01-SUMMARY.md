## Summary

Replaced the blocking `POST /subtitles` + `GET /progress` API pattern with a full SSE async job pattern. The API now immediately returns a `job_id` and streams progress via Server-Sent Events.

## Key Files

### Modified
- `gensubtitles/api/routers/subtitles.py` — Complete rewrite: added `_jobs` dict, `_jobs_lock`, updated `_set_progress` to accept `job` kwarg, added `_run_pipeline_job` background thread function with 3 cancellation check points, added `_cancel_job` helper, 4 new endpoints.

## Endpoints Added

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/subtitles/async` | POST | Start job, return `{job_id}` immediately |
| `/subtitles/{job_id}/stream` | GET | SSE stream of progress events |
| `/subtitles/{job_id}/result` | GET | Fetch completed SRT file |
| `/subtitles/{job_id}` | DELETE | Signal cancellation |

## Endpoints Retired

- `POST /subtitles` — blocking single-request pattern removed
- `GET /progress` — polling replaced by SSE stream

## Decisions

- Used `queue.Queue` per job for thread-safe event passing to async SSE generator
- Used `threading.Event` per job for cancellation signalling
- `_run_pipeline_job` has 3 cancel check points: after extraction, after transcription, after translation
- Error labels truncated to 200 chars (class name + message), not raw exception strings
- Module-level `_progress`/`_progress_lock`/`_set_progress` retained for backward compat
