# Phase 999.9: GUI — Form Polish - Context

**Gathered:** 2026-04-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Three bundled UI micro-improvements to the CustomTkinter form:
1. Add a visual `*` marker to mandatory field labels (input video, output SRT)
2. Hide the source language field — Whisper auto-detection is sufficient
3. Replace the target language free-text entry with a `CTkOptionMenu` dropdown offering "Spanish", "English", and "Other"; "Other" reveals a free-text entry for typing a language code

This phase only modifies `gensubtitles/gui/main.py`. No API, pipeline, or CLI changes.

Implementation must be done in a **new git branch created from remote** (not from the current local branch head).

</domain>

<decisions>
## Implementation Decisions

### Mandatory Field Markers
- **D-01:** Append `*` directly to the label text for the two mandatory fields:
  - `"Input video:"` → `"Input video *:"`
  - `"Output SRT:"` → `"Output SRT *:"`
  No new widgets. Pure label text change.

### Source Language — Hiding
- **D-02:** Hide both the source language `CTkLabel` and `self._entry_source_lang` from the grid via `.grid_remove()` called immediately after their `.grid()` calls in `_build_ui`. Both widgets remain in memory. The existing disable/enable cycle in `_on_generate`/`_finish_generate` can leave them as-is (hidden widgets accept state changes without visible effect).

### Target Language — Dropdown Widget
- **D-03:** Replace `self._entry_target_lang` (`CTkEntry`) with a `CTkOptionMenu`. Widget is selection-only (no free-text typing) — prevents invalid input. Keep `self._target_lang_var` (`ctk.StringVar`) as the `variable` kwarg for the option menu.
- **D-04:** Dropdown option list: `["Spanish", "English", "Other"]`. Friendly display names mapped to API language codes via a module-level dict: `_LANG_MAP = {"Spanish": "es", "English": "en"}`. The API receives the mapped code (or the free-text value when "Other" is selected — see D-05).
- **D-04b:** Default selection on launch: **`"Spanish"`** (pre-selected, `_target_lang_var` initialized to `"Spanish"`).

### Target Language — "Other" Free-Text Entry
- **D-05:** When `"Other"` is selected from the dropdown, a `CTkEntry` appears below the dropdown (new grid row, inserted between the dropdown row and the Generate button row). When `"Spanish"` or `"English"` is selected, the entry is hidden via `.grid_remove()`. The value typed in this entry is passed as `target_lang` to the API. A new `StringVar` (`self._target_lang_other_var`) drives this entry.
- **D-06:** The "Other" entry participates in the disable/enable cycle: disabled in `_on_generate`, re-enabled in `_finish_generate` alongside the other widgets.

### API Call Integration
- **D-07:** In `_on_generate`, determine `tgt_lang` as follows:
  - If selected value is `"Spanish"` or `"English"`: use `_LANG_MAP[selected]` (e.g., `"es"`)
  - If selected value is `"Other"`: use `self._target_lang_other_var.get().strip() or None`
  - The existing `params["target_lang"] = tgt_lang` logic is unchanged; passing `None` means no translation

### Agent's Discretion
- Whether to extract a helper method (e.g., `_on_target_lang_change`) for the "Other" entry show/hide logic vs inlining in the `CTkOptionMenu` `command` callback.
- Whether the Clear button resets the dropdown back to `"Spanish"` (and hides the "Other" entry). The `_target_lang_var` always has a value so it doesn't affect the "has content" check — clearing it is optional.
- Exact row numbers after grid insertion — the "Other" entry inserts between the target lang row and the Generate button row; all rows below shift accordingly.
- Whether `_update_clear_state` checks `_source_lang_var` (which is now always empty/hidden) — can be removed from the check or left as a no-op.

</decisions>

<code_context>
## Existing Code Insights

### Current Grid Layout (after phases 999.1–999.4)
- Row 0: "Input video:" label + `_entry_input` + "Browse…" button
- Row 1: "Output SRT:" label + `_entry_output` + "Save as…" button
- Row 2: "Source language:" label + `_entry_source_lang` (both will be hidden)
- Row 3: "Target language:" label + `_entry_target_lang` (entry → replaced with `CTkOptionMenu`)
- Row 4: Generate button
- Row 5: Clear button
- Row 6: Progress bar (hidden at rest via `grid_remove`)
- Row 7: Stage label
- Row 8: Open Folder button (dynamic, created in `_show_success`)

After this phase, a new row inserts between Row 3 and Row 4 for the "Other" free-text entry (hidden at rest), shifting Generate to Row 5 and everything below down by one.

### Reusable Patterns
- `.grid_remove()` / `.grid()` show/hide: established for `_progress_bar` — same pattern for source lang widgets (always hidden) and the "Other" entry (conditionally shown)
- `self._btn_generate.configure(state="disabled")` / `state="normal"` in `_on_generate` / `_finish_generate` — extend to `_entry_target_lang_other`
- `_target_lang_var = ctk.StringVar()` — repurpose as the `variable` kwarg for `CTkOptionMenu`; `_target_lang_var.get()` returns the selected display name
- `CTkOptionMenu(self._frame, values=[...], variable=self._target_lang_var, command=self._on_target_lang_change)` — standard CustomTkinter pattern

### Integration Points
- `_build_ui`: change label text for rows 0 and 1; hide row 2 widgets; replace row 3 entry with `CTkOptionMenu`; add new "Other" entry row (hidden initially)
- `_on_generate`: add `_entry_target_lang_other.configure(state="disabled")`; compute `tgt_lang` using `_LANG_MAP` + "Other" branch
- `_finish_generate`: add `_entry_target_lang_other.configure(state="normal")`
- Module level: add `_LANG_MAP = {"Spanish": "es", "English": "en"}`
